基于工单信息和知识库来源，生成处理建议 JSON：
{
  "summary": "工单摘要",
  "possibleCauses": ["可能原因"],
  "actions": ["立即处理动作，偏短句"],
  "runbookSteps": ["可执行 Runbook 步骤，按顺序，从确认现象到验证修复"],
  "additionalInfoNeeded": ["待确认信息"],
  "risks": ["风险提示"],
  "sources": ["来源ID"]
}
要求：
1. 只能基于工单、根因分析、Tool 取证结果和知识库来源，不编造根因。
2. actions 面向当前值班人员，强调立即止血和关键排查。
3. runbookSteps 必须可执行，格式建议为 "[Step 1] ..."。
4. 不确定信息放入 additionalInfoNeeded。
5. sources 只填写输入中真实出现的来源 ID。

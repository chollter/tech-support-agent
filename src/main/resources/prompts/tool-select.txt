请基于 AgentPlan、可用工具白名单和工单内容，选择需要执行的工具，并填写工具参数。

输出 JSON（不要 markdown 代码块）：
{
  "selectedToolNames": ["query_metric", "query_logs"],
  "parameters": {"system": "payment-service", "module": "/pay/callback"},
  "reason": "OOM 场景查内存指标和 Pod 日志"
}

规则：
- selectedToolNames 只能来自白名单：searchSimilarCases、query_logs、query_metric
- 必须与 AgentPlan 中未 skipped 的工具类步骤一致
- AgentPlan 中 skipped 的步骤对应工具不要选
- KNOWLEDGE_SEARCH 不是 Tool，不要出现在 selectedToolNames
- 不要编造工具名

parameters 参数填写规则：
- system：受影响的系统名（如 payment-service）。若工单未明确，从上下文推断；推断不出则不填
- module：受影响的模块/接口（如 /pay/callback）。同上
- 填写的参数会覆盖已抽取字段——所以只在确信工单信息更准确时才填
- 查询范围越精确（system+module 都填），日志/指标结果越聚焦
- searchSimilarCases 用语义检索，无需填 system/module
